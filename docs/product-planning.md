# Trust or Trap — 기획안 (AI 참조용 요약)

> 이 문서는 원본 기획안을 **AI 에이전트가 빠르게 이해**하도록 압축한 참조본입니다.
> 구현 규칙은 `.cursor/rules/`, 작업 절차는 `.cursor/skills/`, 용어는 [domain-glossary.md](domain-glossary.md)를 함께 참조하세요.

---

## 1. 한 줄 정의

실제 스마트폰 환경("폰 속의 폰" UI)을 재현한 **몰입형 인터랙티브 피싱/디지털 스캠 예방 시뮬레이션**. 사용자가 정답을 외우는 것이 아니라, 현실과 유사한 상황에서 **위험 신호를 스스로 식별하고 판단**하도록 훈련한다.

## 2. 문제 정의 & 차별화

- 기존 예방 콘텐츠는 **객관식 정답 학습** 중심 → 실제 상황의 판단력 훈련이 어렵다.
- 실제 스캠은 문자/전화/AI 사칭 등 형태가 다양하고 갈수록 교묘하다.
- **차별화 3가지**
  1. 사용자 입력(나이·성별)으로 **발생 확률이 가장 높은 범죄 스토리**를 매칭하는 스토리형 시뮬레이션.
  2. **"폰 속의 폰" UI** — 실제 앱과 동일한 인터페이스를 재현해 "게임"임을 숨기고 경각심을 준다.
  3. **AI 기반 개인 맞춤 시나리오** — 정답형이 아닌 판단형 학습.

| 연령대 | 여성 (기본 매칭) | 남성 (기본 매칭) |

## 4. 전체 사용자 플로우

```
[1] 웹 접속 → 이름(가명 가능) / 나이 / 성별 입력
[2] 페르소나 매칭 (매트릭스 기반 자동 추천)
     "OO님, 최근 당신 또래·성별에서 가장 많은 사건은 [매칭 시나리오]입니다."
     → [바로 시작] 또는 [다른 사건 선택하기]
     → 투자리딩방 등 전 연령·성별 대상 시나리오는 항상 선택지에 노출
[3] "폰 속의 폰" 진입 → 홈 화면 → 앱 아이콘 배치 → 실시간 알림으로 스토리 시작
[4] 인터랙션 → 객관식이 아닌 "지금 뭐라고 답할지" 자유 선택/입력
     → AI가 응답의 위험도(risk_flag) 판정 → 다음 노드 분기
     → 시간 압박 요소(타이머) 부여
[5] 엔딩 & 리포트 → 결말 3갈래 + "놓친 위험 신호" 리플레이 리포트
```

플로우의 세부 분기(각 시나리오별 노드 흐름)는 원본 기획안 4-1 mermaid 다이어그램을 시나리오 그래프 JSON으로 변환해 구현한다.

## 5. 엔딩 3갈래 (모든 시나리오 공통 구조)

| --- | --- | --- |
| ✅ 안전(safe) | 위험 신호 식별 후 회피/신고 | 안전 종료 |
| ⚠️ 경고(warning) | 소액 피해 후 인지·대응 | 신고/지원기관 절차 안내 |

## 7. 페르소나별 시놉시스 & 위험 신호 학습 포인트

각 시나리오는 **`접근 → 신뢰형성 → 위험신호 제시 → 요구 → 분기(안전/경고/피해)`** 공통 골격을 따른다.

### ① 10대 여성 — 온라인 그루밍 `teen-female-grooming`
- 흐름: SNS DM 접근·고민상담 → 친밀감 형성(평균 ~102일) → 화상통화/사진 요구 → 협박 전환.
- 학습 포인트: 급격한 친밀감 속도, "부모/선생님한테 비밀" 요구, 사진 요구의 단계적 확대.
- 특수 규칙: **음성(TTS/STT) 미적용 — 텍스트/이미지 채팅형만.** "며칠째 대화 중" 경과일 표시로 장기성 체감.

### ② 10대 남성 — 게임 아이템 사기 / 몸캠피싱 `teen-male-gameitem`
- 흐름: 저가 아이템 판매 제안 → 가짜 후기·인증샷 → 선입금 요구 or 영상통화 유도 후 몸캠 협박.
- 학습 포인트: 첫 거래 상대와의 선입금 구조 위험성, 안전거래(에스크로) 부재 확인법, 영상통화가 사적으로 흐르는 경계 신호.
- 특수 규칙: **음성 미적용.**

### ③ 20대 여성 — 로맨스 스캠 `twenties-female-romance`
- 흐름: 데이팅앱 매력적 프로필 → 수 주 정서적 유대 → 환전사기 위기(가장 흔한 유형, 55.4%) → 금전 요청.
- 학습 포인트: 화상통화 회피 이유, 프로필 사진 역이미지 검색, "사랑하니까 도와달라" 감정 프레이밍.
- 분기 특징: 송금 전 **"실존 확인" 미니 조사**(프로필 검색/통화 요청)를 넣어 조사 행동 유도.

### ④ 20대 남성 — 알바위장 대출빙자형 보이스피싱 `twenties-male-voicephishing`
- 흐름 A(대출빙자): 저금리 대환대출 문자 → 원격제어 악성앱 설치 → "먼저 상환해야 신용점수 상승" → 송금 압박.
- 흐름 B(알바사기): 통장·체크카드 양도 → 본인도 모르게 대포통장 유통 가담(**형사처벌 리스크** 체험이 핵심).
- 학습 포인트: 금융기관은 원격제어 앱을 요구하지 않음, "선상환 후 대출" 구조의 비정상성.

### ⑤ 30~40대 — 투자리딩방 `middle-invest-scam`
- 흐름: 오픈채팅방 초대 → 허위 수익 인증 도배 → 가짜 HTS 소액 성공체험 → 고액 유도 → 인출 시 수수료·세금 명목 추가 입금 → 잠적.
- 학습 포인트: 원금·고수익 보장 문구, 제도권 등록 여부(금감원) 확인법, **인출 시점마다 새 명목 추가** 패턴.

### ⑥ 50대 — 대환대출·정책자금 빙자형 보이스피싱 `fifties-loan-scam`
- 흐름: 정책자금·저금리 대환 문자/전화 → "선상환해야 신용점수 상승" → 지정계좌 선입금 or 보증보험료 명목 요구.
- 학습 포인트: 선입금 요구 구조의 비정상성, 금감원(1332) 실시간 확인 습관.

### ⑦ 60대+ — 기관사칭형 보이스피싱 `senior-authority-scam`
- 흐름: 검찰/금감원/관세청 사칭 → "당신 계좌가 범죄 연루" 공포 조성 → 가짜 정부 사이트 → "안전계좌" 이체 요구. 자녀 사칭 병행 가능.
- 학습 포인트: 수사기관은 전화로 계좌 이체를 요구하지 않음, 자녀 사칭 시 본인만 아는 질문으로 재확인, 통화 중 별도 확인 전화.

## 8. 통계 근거 (콘텐츠 리얼리즘 반영)

시나리오 대사·수치·연출은 아래 실데이터를 반영해 경각심을 강화한다(전체 통계 및 출처는 원본 기획안 3·5절 참조).

- **로맨스 스캠**: 피해자 여성 71.4%, 30대 이하 87.5%, 환전사기 55.4%.
- **보이스피싱**: 기관사칭형 60대 단독 비중 30.6%(1위), 대출빙자형 40~60대 90% 집중, 기관사칭형 20대 이하도 40.8%로 양극화.
- **투자리딩방**: 20~70대 전 구간, 공통 구조 = "전문가 사칭 + 허위 수익 인증 + 인출 시 추가 입금 요구".
- **디지털 성범죄**: 10·20대 77.6%, 아동 피해 평균 13.9세, 접근 경로 채팅앱 45.0%, 그루밍 평균 102.4일.

> ⚠️ 로맨스스캠·투자리딩방은 국가 전수통계가 없어 표본/사례 기반 수치임. 보이스피싱·디지털성범죄는 공식 통계.

## 9. 콘텐츠 안전 원칙 (매우 중요)

- 미성년자 대상 시나리오(10대)는 **실제 성적 콘텐츠·노골적 묘사를 절대 생성하지 않는다.** 위험 "구조"만 학습시킨다.
- 실존 인물·실제 기관의 사칭 대사는 생성 금지(가상 명칭 사용).
- 상세 안전 규칙: `.cursor/rules/content-safety-teen.mdc` 및 [technical-architecture.md](technical-architecture.md) 참조.
